# 모노레포 아키텍처 (Monorepo Architecture)

## 개요
기존의 단일 NestJS 애플리케이션 구조에서 벗어나, 3개의 독립적인 서비스를 한 곳(레포지토리)에서 효율적으로 관리하기 위해 **Monorepo(모노레포)** 아키텍처로 전환했습니다.

## 프로젝트 구성
현재 모노레포는 아래의 구조로 분리되어 있습니다:

- **`apps/mesapphub`** (사내 관리용 메인 백엔드 분리 - 기본 프로젝트 5107번 포트)
- **`apps/oms-api`** (학회 OMS 서비스 API - 5108번 포트)
- **`apps/intos-api`** (인토스 앱 모바일 API - 5109번 포트)
- **`libs/shared-core`** (3개의 앱이 공유하는 공통 라이브러리)

## 주요 장점
1. **코드 재사용성 극대화:** 로거, 인증, 미들웨어, DB 설정 등 공통 로직을 `shared-core` 라이브러리로 분리하여 각 API 앱들이 `@app/shared-core` 패키지 이름으로 무한 재사용할 수 있습니다.
2. **독립적 배포 지원:** 하나의 소스코드 저장소를 가지고 각 앱별로 컴파일(`npm run build <app>`)하고, 이 과정을 Dockerfile의 `ARG APP_NAME`으로 묶어 단 하나의 도커 템플릿으로 3개의 각기 다른 이미지 빌드를 지원합니다.
3. **공통 부트스트랩:** 3개의 `main.ts`가 각자 복잡한 설정을 하던 문제를 해결하고, `shared-core`의 `configureSharedCore()` 함수 한 줄만으로 Swagger, Helmet 등 수십 가지의 서버 기본 옵션이 모든 앱에 똑같이 적용됩니다.
