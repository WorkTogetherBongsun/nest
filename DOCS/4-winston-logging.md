# Winston 데일리 파일 로깅 시스템 (Daily Rotate Logging)

## 개요
애플리케이션 운영에 치명적인 이슈를 파악하거나 일 단위 지표를 확인하기 위해 Node.js 환경에서 가장 성숙한 로거 패키지인 **`winston`**과 **`winston-daily-rotate-file`**을 채택했습니다. 

## 로깅 구조 (공유 팩토리 함수)
모노레포의 장점을 활용하여 3개의 각각 다른 앱(mesapphub, oms-api, intos-api)이 `shared-core`의 중앙 집중 함수(`getSharedLogger.ts`)를 호출하며 앱 이름만 전달하여 동일한 표준의 로그 정책을 부여받습니다.

### 적용되는 로그 정책:
1. **분리 저장 전략:** 애플리케이션의 최상단 디렉토리 안에 `LOG/{appName}/` 형태로 자동 격리되어 파일이 쌓입니다.
2. **에러 로깅 독립 (Error Level):** `info`나 `debug` 잡스레기 로그에 중요한 에러(오류)로그가 묻히는 것을 방지하기 위해 에러만 저장하는 `[date].error.log` 파일이 별도 생성됩니다.
3. **압축 보관 (ZippedArchive):** 
   - 매일 자정 혹은 용량이 초과하면 옛 로그는 스스로 압축(Zip)하여 파일 시스템의 디스크를 아낍니다.
4. **리텐션 기간 (보관 주기):** 
   - 에러 로그는 트래킹/추적을 위해 **30일** 보관
   - 기본 작동 로그(`info` 이상)는 **14일**만 보관하고 오래된 로그부터 자체 삭제하도록 설정 (하드 용량 터짐 방지 설계 채택)

## 동작
이제 `console.log` 대신 앱 전반에 걸쳐 NestJS 표준의 `Logger`를 호출하게 되면 화면과 디스크 양쪽에 동시/안전 저장됩니다.
